# 📂 Проект: DystopiA! (Core DA-21.8)

Технический паспорт архитектуры и текущего прогресса разработки.

---

## 🏛 1. Глобальная архитектура (Backend)
Система построена на принципах **Data-Driven** и **Event-Driven** архитектуры с использованием паттерна Singleton (Autoload).

### **SettingsManager (Autoload: `Settings`)**
Центральный системный узел (`Node`), управляющий состоянием игры.
- **Инкапсуляция**: Данные разбиты на вложенные менеджеры (`SettingsManager_Volume`, `SettingsManager_Locale`).
- **Персистентность**: Зашифрованное хранилище (`user://settings.dat`) с использованием `_SALT`.
- **Реактивность**: Каскадные сигналы. Вложенные менеджеры сигнализируют родителю, который вещает в общую шину `is_changed(type)`.
- **Защита**: Реализован **Debounce** (отложенное сохранение через 3 сек) и «Шлюз» (`_EMITTER`), отключающий сигналы во время загрузки JSON.

### **Event Bus (Autoload: `ESB`)**
Kafka-подобная шина для слабой связанности компонентов.
- Метод `publish(topic, data)` позволяет атомам UI и системным сервисам общаться без прямых ссылок.

---

## 🔊 2. Аудио-движок
Интеграция с системным `AudioServer` Godot.
- **Иерархия шин**: `Master` ← (`Music`, `Sound`, `Voice`, `Ambience`).
- **Математика**: Перевод из `linear (0-100)` в `db` с безопасным порогом тишины `-80.0 dB` (вместо `-inf`).
- **Авто-микширование**: `SettingsManager` автоматически обновляет громкость шин при каждом изменении через метод `_sync_audio_server`.

---

## 🧬 3. UI-Фреймворк (Angular-style)
Система динамической генерации интерфейса из конфигурационных объектов.
- **Концепция**: UI не рисуется в редакторе, а строится `FormBuilder`-ом на основе JSON/Dictionary.
- **Компоненты (Атомы)**: Независимые сцены (`FieldSlider`, `FieldButton`), реализующие метод `setup(data)`.
- **Синхронизация**: Атомы связаны с `Settings` через строгую типизацию и реактивные сеттеры.

---

## 🛠 4. Технические стандарты проекта
- **Типизация**: Обязательное использование `as Dictionary`, `clampi`, `clampf` для предотвращения деградации данных.
- **Feature Tags**: Использование `OS.has_feature("editor")` для автоматической активации `DEBUG` режима.
- **Код-стайл**: 
  - `class_name` — PascalCase (в начале файла).
  - Методы и переменные — snake_case.
  - Приватные члены — префикс `_`.
  - Read-Only свойства в Autoload защищены через `set(value): pass`.
